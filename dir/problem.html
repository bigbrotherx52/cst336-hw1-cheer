The "Foam"

You're developing for a console, or developing software for a PoS that is meant to be affordable and light. chances are better then not you're going to run into memory limitations. Why throw 8 gigs of ram into what can accomplish the same task with just 500mb? A potential issue this could cause is that when you load something into memory, and then something else, deallocating that first chuck of memory will leave the second one floating. Now imagine this going on for a while, eventually it's possible your memory starts to look "foamy" and it becomes more and more difficult to find places to put your data.

This foamy memory will result in a crashed machine. Games released to consoles (which might not have as well regulated memory depending on platform and software) nowadays will be required to go through several days of idling to see if it will crash.

Another issue is loading and unloading objects to memory can be a bottle neck, specially when that object might need to be loaded again soon after it was removed. A solution to both these problems can be found in the Object Pool!
