So what is an object pool?

Object pools are a type of data structure that relies on you just throwing your objects into memory, and keeping them there. The example given here is one based off a simple particle system for a game. There are 2 elements: the object, and the object pool. the pool holds an array of the objects, in this case the particle class, all created as dead, unused elements. When they are needed, the user will call on the pool to get a new one to use. this could be achieved any number of ways. Once the object has been create, the user will again call on the pool to do something to all the objects (or can call each object themselves). When the object is finished, it is turned off, and returned to the off state, where it can be called again later on.

an issue that might be apparent to some observers that it might take a while to find the next available object to be used. This is where Free Lists come into play.
